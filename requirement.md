请基于**前后端分离架构**，从零到一完整开发一款**本地视频播放管理平台**，严格遵循以下需求、技术栈和开发要求，输出可直接落地的完整方案+代码实现：

## 一、产品定位与目标用户
1. **核心定位**：轻量化、本地优先的视频播放+资产管理平台，主打学习视频管理，无冗余功能，部署简单、本地运行流畅
2. **目标用户**
   - 学生群体：用于播放本地学习视频、上传课程视频，管理学习资料
   - 企业/社团小团体：用于内部学习培训、教材视频统一管理与播放

## 二、核心功能需求（必实现）
### 1. 视频来源管理
- 支持**本地文件夹直接导入**视频（自动扫描、读取本地视频文件）
- 支持**用户手动上传视频**到平台本地库
- 兼容主流视频格式（MP4、AVI、MKV、MOV、FLV等）

### 2. 视频播放核心
- 原生流畅播放本地/上传视频
- 基础播放控制：暂停、快进/快退、倍速、音量、全屏
- **播放记录自动保存**：记录最后播放进度、播放时间、播放次数

### 3. 视频资产管理
- 视频收藏功能：收藏/取消收藏，收藏列表独立展示
- **视频标签系统**：支持为单个视频添加多个自定义标签，按标签筛选、检索视频
- 基础视频信息展示：封面、文件名、时长、格式、上传/导入时间

### 4. 基础交互
- 视频列表/网格视图切换
- 搜索功能：按视频名称、标签搜索
- 播放记录、收藏、标签数据持久化存储

## 三、指定技术栈（严格遵守）
### 前端
- 构建工具：Vite
- 框架：React
- UI组件：ShadCN UI
- 样式：TailWindCSS 4
- 能力：本地文件读取、视频播放、前端状态管理

### 后端
- JDK版本：JDK 21
- 核心框架：Spring Boot
- 数据库：开发环境H2DB（内存/文件型，本地免部署）、生产环境MySQL
- 数据持久化：Spring Data JPA
- 核心能力：本地文件扫描、视频元数据管理、播放/收藏/标签数据接口

### 架构
- 标准前后端分离
- 本地优先运行，支持单机部署

## 四、开发要求
1. 从零搭建项目骨架，包含前端工程+后端工程完整目录结构
2. 输出完整可运行代码，无语法错误、可直接本地启动
3. 接口文档清晰，前后端联调逻辑完整
4. 界面简洁美观，符合学习工具使用场景，操作极简
5. 适配本地运行，无需复杂配置，开箱即用

修改任何的代码后，都需要重新编译和重启项目，才能生效。
后端java修改后，编译的命令为：`mvn clean install`
前端修改后，编译命令为：`npm run build` 

编译过后的项目，都需要检查是否需要重启服务器，重启服务器的命令为：
- 后端java项目：
    - 发现端口是否被占用的命令为： `lsof -n -P -iTCP:8080 -sTCP:LISTEN`
    - 发现端口被占用，需要先终止占用进程，命令为：`kill -9 进程ID`
    - 终端重新启动服务器，命令为：`mvn spring-boot:run`
- 前端项目：
    - 发现端口是否被占用的命令为： `lsof -n -P -iTCP:5173 -sTCP:LISTEN`
    - 发现端口被占用，需要先终止占用进程，命令为：`kill -9 进程ID`
    - 终端重新启动服务器，命令为：`npm run dev` 
注意不要随意修改端口号，否则会导致项目无法正常运行。
- 前端项目端口号为：`5173`
- 后端java项目端口号为：`8080`
